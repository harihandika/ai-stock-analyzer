# Agile Sprint Plan — AI Stock Analyzer

Pengembangan AI Stock Analyzer ini dibagi menjadi **5 Sprint** dengan asumsi 1 Sprint = 1–2 minggu kerja.
Setiap Task memiliki deskripsi fitur wajib, acceptance criteria, dan file output yang diharapkan.

## Sprint 3: Advanced VPA & Smart Money Concepts ✅ DONE

**Objective**: Membangun algoritma matematika non-AI untuk mendeteksi Wyckoff Phase dan Smart Money Concepts.

| Task | Fitur Wajib | Acceptance Criteria | Output File |
|------|-------------|---------------------|-------------|
| **3.1** Wyckoff Phase Engine | Implementasi fungsi `detect_wyckoff_accumulation(df)`. Deteksi **Phase C (Spring)**: harga mencetak new low 50 hari + rejection kuat (close_pos > 0.5) + volume > 1.2x VMA. Deteksi **Phase D (Sign of Strength/SOS)**: candle bullish + volume > 1.5x VMA + terjadi dalam 15 hari setelah Spring. Output: kolom `is_spring` (boolean), `wyckoff_phase` (string: "Phase C (Spring)" / "Phase D (SOS)" / None) | Pada data simulasi dengan Spring di index 60, kolom `wyckoff_phase` bernilai "Phase C (Spring)". Pada SOS 5 hari setelahnya, bernilai "Phase D (SOS)" | `app/services/wyckoff_engine.py` |
| **3.2** SMC: Fair Value Gap (FVG) | Implementasi fungsi `detect_fvg(df)`. **Bullish FVG**: Low candle(t) > High candle(t-2). **Bearish FVG**: High candle(t) < Low candle(t-2). Output: kolom `bullish_fvg` (boolean), `bearish_fvg` (boolean), `fvg_size` (float, ukuran gap) | Pada data dimana harga melompat naik tajam, `bullish_fvg` bernilai True dan `fvg_size` > 0 | `app/services/smc_engine.py` |
| **3.3** SMC: BOS & CHoCH | Implementasi fungsi `detect_structure_breaks(df)`. Memerlukan Swing Points dari Sprint 2. **BOS**: Close menembus Swing High/Low searah tren yang sedang berjalan. **CHoCH**: Close menembus Swing High/Low berlawanan arah tren (sinyal pembalikan). State machine sederhana untuk tracking tren saat ini (Uptrend=1, Downtrend=-1, Sideways=0). Output: kolom `bos` (boolean), `choch` (boolean) | Pada data simulasi uptrend yang menembus swing high sebelumnya, `bos` bernilai True. Pada pembalikan arah, `choch` bernilai True | `app/services/smc_engine.py` (di dalam file yang sama) |
| **3.4** Integrasi ke Endpoint `/sync` | Update endpoint `POST /stocks/{ticker}/sync` untuk memanggil `detect_swing_points()`, `detect_wyckoff_accumulation()`, `detect_fvg()`, dan `detect_structure_breaks()` secara berurutan. Hasil SMC dikumpulkan ke dictionary JSON (`bullish_fvg`, `bearish_fvg`, `fvg_size`, `bos`, `choch`, `swing_high`, `swing_low`) dan disimpan ke kolom JSONB `smc_patterns`. `is_spring` dan `wyckoff_phase` disimpan ke kolom masing-masing di `TechnicalIndicator` | Hit endpoint `/sync` dan query DB: kolom `smc_patterns` berisi JSON valid, `wyckoff_phase` terisi saat ada Spring | `app/api/routers/stocks.py` (modifikasi) |
| **3.5** Unit Test (Wyckoff + SMC) | Test Spring detection: simulasi new low + rejection + high volume → `is_spring == True`. Test SOS detection: simulasi bullish candle 5 hari setelah Spring → `wyckoff_phase == "Phase D (SOS)"`. Test Bullish FVG: data lompatan harga → `bullish_fvg == True` dengan fvg_size benar. Test Bearish FVG: data penurunan tajam → `bearish_fvg == True`. Test CHoCH: data pembalikan tren → `choch == True`. Test empty DataFrame → tidak crash | Semua test `PASSED` | `tests/services/test_wyckoff_engine.py`, `tests/services/test_smc_engine.py` |
